## 📖 文章一：Anthropic Multi-Agent 分工實驗
**核心發現**：一個 AI 做不好的事，拆成三個 AI (Planner → Generator → Evaluator) 就做得好了。

| 文章洞察 | ConditionAI 可優化應用 | 對應部門 |
|---------|----------------------|---------|
| AI 自己 review 自己會「自我感覺膨脹」 | Agent 2 產出結果**絕對不能由 Agent 2 自己驗收**，必須交給獨立的 Agent 3 與紅軍做交叉審查 | 稽核部 + 法務部 |
| Generator 與 Evaluator 簽「Sprint Contract」 | n8n Agent 1→2→3 之間加入**格式約束契約**：不符 JSON Schema 就退回重做 | 資料部 |
| Context Window 塞滿導致品質崩壞 | 長篇報告應**拆分為多輪 Sprint**，每輪只處理一個身體區域 | 研發部 |
| 模型進步後要「拆除鷹架」 | 定期回顧 Prompt，LLM 升級後刪除過時防呆指令，**降低 Token 成本** | 研發部 + 稽核部 |

| 文章洞察 | ConditionAI 可優化應用 | 對應部門 |

## 📖 文章二十三：團隊 AI 化三階段（流程化 → 數位化 → AI 化）
