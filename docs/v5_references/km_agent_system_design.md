# ConditionAI

## 知識管理 Agent 系統設計計畫書
版本 V1.1　｜　2026 年 3 月　｜　內部設計文件

# 一　系統定位與設計目標
不只在程式碼，更在創辦人累積的臨床知識。任務是將知識轉化為結構化、可被 AI 動態注入的知識庫。
目標：知識快速入庫 (< 60 秒)、知識品質把關 (< 2 分鐘人工決策)、知識即時應用 (命中率 ≥ 60%)。

# 二　三角色分工架構
1. **師傅端**：貼文提供知識，提交回饋。
2. **審核者（管理層）**：最終確認，標籤批准。
3. **系統（Multi-Agent n8n）**：自動萃取、合規審核、整合報告。

# 三　Multi-Agent 審核流水線
- **Agent 1 資料蒐集員**：萃取知識片段，一文多穴自動批次拆分。
- **Agent 2 結構化整理員**：對齊 `tag_library`，產生草稿 JSON。
- **Agent 3 合規審查員**：20 個禁止詞彙防護，衝突偵測。
- **Agent 4 報告整合員**：彙整重點供人類審核者決策。

# 四　資料表與點數經濟
- **knowledge_base**：新增 `applicable_techniques`、`contraindications`、`source_credibility` 等欄位。
- **tag_library**：動態標籤管理。
- **credits**：貢獻知識獲得點數，用於折抵月費，達成雙時間點學習迴圈。

# 五　RAG 知識應用
1. 禁忌交叉比對 (Contraindications filtering)。
2. 師傅專長過濾。
3. 可視化：在客戶建議結尾附註「參考足五里 LR10 知識，由臨床經驗萃取」。
