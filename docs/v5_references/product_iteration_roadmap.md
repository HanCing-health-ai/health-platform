# ConditionAI

## 產品迭代路線圖
各版本交付清單與功能細節說明

| 階段 | 時間 | 目標 |
| --- | --- | --- |
| **V1** | 0-6 個月 | 核心系統建立：讓第一家工作室真實使用 (Risk Triage, 雙向輸出) |
| **V1.5**| 試點驗證後 | 體驗升級：視覺化＋歷史追蹤 (身體圖、回訪時間軸) |
| **V2** | 7-18 個月 | 商業化啟動：LLM 完整版＋平台媒合＋多元收入 (AI Scribe、商品導購) |
| **V3** | 19-36 個月 | 平台化：完整 SaaS＋ESG＋API 開放 (K-means 自動聚類、商城) |
| **長期**| 3 年後 | 產業基礎設施：數據分析與認證標準 |

# 核心開發模組軌跡
- **前端（Next.js 14）**：從文字勾選 (V1) 演進至互動 SVG 身體圖 (V1.5)。
- **後端（FastAPI）**：從嚴格規則分流模型 (V1) 升級為 Claude Structured Output (V2)。
- **資料庫（Supabase）**：包含 clients, questionnaire_responses, insight_reports, tag_library，並貫徹 RLS 隔離。
- **商業模式擴張**：工具月費 → 商品導購 / 預約抽成 (V2) → ESG 報告、線上課程、API 開放 (V3)。

**跨版本持續性工作**
- 安全維護：API Key 輪換、Git 安全檢查。
- 合規審查：每季語言審查，確保無醫療用語滲透。
- 資料飛輪原則：每一代版本累積的資料都是下一代的基礎，不可跳階。
