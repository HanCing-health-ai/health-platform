# triage-logic
description: ConditionAI 風險分流邏輯說明。實作分流相關功能時自動載入。

## 三類分流定義
- A 類：正常通過，呼叫 Claude API 進行分析
- B 類：警示，顯示提醒訊息，仍可進行分析但標記低信心
- C 類：阻斷，不呼叫 Claude API，顯示建議就醫訊息

## C 類阻斷邏輯
來源：triage_keywords.json
判斷層：FastAPI 後端（不在前端做）
阻斷方式：return 固定訊息，不呼叫 Claude API
阻斷訊息範本：「您描述的狀況建議先諮詢專業醫療人員」

## B 類處理邏輯
信心分數 < 閾值時標記為「待師傅確認」
仍呼叫 Claude API，但在師傅端加上確認提示

## 重要原則
- 分流判斷在後端執行，前端只顯示結果
- 師傅保有最終決策權，分流結果是建議不是指令
- 嚴禁在分流訊息中使用醫療用語
