# ConditionAI 標準分層管理 (Standards Registry)
**不可變的規則 vs. 可調整的政策，必須分開管理。**

---

## 🔒 不可變規則 (Immutable Process Rules)
**這些規則任何人、任何聊天室、任何版本都不能修改。修改它們等於破壞系統的底層安全。**

### 合規紅線
- **20 個絕對禁詞**：治療、療效、診斷、藥物、疾病、處方、醫療、治癒、病症、症狀、病理、臨床、醫囑、用藥、病灶、療程、癒合、病變、手術、處置
- 以上禁詞的同義詞、變體寫法、語意暗示同樣禁止
- **兒童相關輸出**：禁止任何暗示生長發育效果、睡眠障礙改善、消化功能改變的表述

### 工程紅線
- 禁止讀寫 `.env` 檔案
- 所有 production code 必須有對應測試（TDD Iron Law）
- Bug 修復不超過 3 次嘗試就必須停手（3-Strike Rule）
- 不可在未跑驗證的情況下宣稱「完成」（Verification Iron Law）

### 資料安全
- 客戶個資不可出現在 AI 訓練資料或日誌中
- 所有資料庫查詢必須通過 RLS（Row Level Security）
- 知識庫條目必須經過合規審查才能設為 `active`

### 角色邊界
- Antigravity 的技術建議 ≠ 最終決策
- 合規語言的最終判斷權歸 Claude 策略端
- 人類 CEO 擁有唯一的商業方向否決權
- **Gemini 只做文章粗加工**：禁止產出架構藍圖、V2 企劃報告、資料表設計、優先順序判斷

## 🔧 可調整政策 (Adjustable Organizational Policies)
**這些政策可以根據專案階段、團隊規模、商業需求調整。調整時記錄原因。**

### 開發流程
| 政策 | 當前設定 | 可調範圍 |
|------|---------|---------|
| 規劃週期 | 90 天衝刺 | 30-180 天 |
| 研發室預設模型 | Gemini 3.1 Pro | 可換更新模型 |
| 策略室預設模型 | Claude Opus 4.6 Thinking | 可換更新模型 |
| 每批次執行任務數 | 3 個 | 1-5 個 |
| Code Review 觸發時機 | 每完成🔴或🟡任務 | 可放寬為每 phase |

### 知識庫營運
| 政策 | 當前設定 | 可調範圍 |
|------|---------|---------|
| 知識貢獻點數 | +10 點/篇 | 依類別動態調整 |
| 回訪週期 | 1-3天 / 28天 / 56天 / 84天 | 可新增或調整間隔 |
| 信心閥值（未定） | 待 Claude 定規格 | 0.5-0.9 |
| n8n Agent 模型 | 全部 Claude Sonnet | 可按任務分級 |

### 團隊與權限
| 政策 | 當前設定 | 可調範圍 |
|------|---------|---------|
| 部門聊天室數量 | 3 個啟用 + 3 個待命 | 按需增減 |
| AI 熟練度門檻 | L1 以上 | 可隨團隊成長提高 |
| 新功能上線流程 | 內部 → Alpha → Beta → GA | 可簡化或加嚴 |

### 商業策略
| 政策 | 當前設定 | 可調範圍 |
|------|---------|---------|
| 商品推薦啟動時機 | V2 後期（等付費用戶） | 可提前或延後 |
| 小兒推拿啟動時機 | V3（等法律顧問） | 可提前（如法規確認） |
| 試點轉付費條件 | 8 週調理報告 | 可調整週期長度 |
